srsh a small shell with a weird little scripting language

srsh is a Ruby-powered interactive shell with RSH scripting: familiar-ish keywords, not Ruby eval, and a focus on staying simple. It’s not trying to replace your real shell, it’s trying to be fun and usable.

Why srsh exists

For people who like tinkering with shells, themes, plugins, and little languages without needing to write C or fight your system shell config.

It’s also a nice playground for RSH: a purposely-limited expression engine and script runner.

Quickstart

Install the gem:

gem install srsh

Run it:

srsh

Change theme:

scheme --list
scheme ocean

RSH basics

RSH scripts are plain text files. srsh reads them, strips comments (# outside quotes), parses blocks, and runs commands line-by-line.

A tiny script:

# hello.rsh
put "hello"
NAME = "world"
emit "hello, " .. $NAME

Expressions

Expressions are parsed by the RSH lexer/parser, not Ruby. Basic operators:

  • math: + - * / %
  • compare: == != < <= > >=
  • logic: and / or / not (also &&, ||, !)
  • concat: .. (string-ish join)

Functions you can call in expressions:

emit len("abc")
emit contains("srsh", "rsh")
emit rand(10)
emit pick("sand", "water", "lava")

Control flow

Blocks end with end. It’s intentionally old-school.

if $USER == "user"
  put "wsg gng"
else
  put "hello"
end

times 5
  emit "it=" .. $it
end

i = 0
while $i < 3
  put "looping"
  i = $i + 1
end

Functions

RSH functions are defined with fn. Arguments become ENV variables inside the function. 

fn greet who
  emit "hello " .. $who
end

greet "world"

Themes

Themes live in ~/.srsh/themes. You can write either a simple .theme file or JSON. 

# ~/.srsh/themes/my.theme
ui_border=1;35
ui_title=1;33
ui_hdr=1;36
ui_key=1;36
ui_val=0;37
ok=32
warn=33
err=31
dim=90
prompt_path=33
prompt_host=36
prompt_mark=35
quote_rainbow=true

Then:

scheme my

Plugins

Put .rsh or .rb files in ~/.srsh/plugins. RSH plugins run first (safer), Ruby plugins run second (trusted). 

# ~/.srsh/plugins/hello.rsh
alias ll='ls'
put "srsh plugin loaded"

A tiny Ruby plugin can register a builtin via the SRSH API:

# ~/.srsh/plugins/hi.rb
SRSH.builtin("hi") do |_args|
  puts "hi from ruby plugin"
end

That’s the basics. If you break it, you get to keep both pieces :D

About srsh

The goal: small, hackable, and not scary to modify.

What it is

srsh is an interactive shell written in Ruby. It includes a separate scripting language called RSH with its own lexer/parser and an expression evaluator.

It intentionally avoids Ruby eval in scripts. That keeps RSH predictable and limits “oops I executed a string” moments.

What it isn’t

  • Not a full POSIX shell replacement
  • Not trying to be bash/zsh/fish
  • Not doing fancy job control yet
  • Not a “secure sandbox” (Ruby plugins are trusted)

About Ruby

srsh is built with Ruby - the language is doing the heavy lifting.

Ruby is a general-purpose language with a strong ecosystem (gems), good text handling, and a “make it nice” vibe. srsh uses Ruby for the shell itself, but RSH stays separate so scripts don’t become random Ruby code.

If you want to learn Ruby basics, a good start is the official Ruby docs and RubyGems. (This site is styled like RubyGems on purpose, it’s a nice layout.)
